# Pterodactyl-Nodejs-egg
HaZeyHosting

L’egg NodeJS pour Pterodactyl permet d’exécuter des applications Node.js au sein du panel de gestion de serveurs Pterodactyl.
L’une de ses fonctionnalités principales est la possibilité de définir une **commande de démarrage personnalisée**, offrant ainsi davantage de flexibilité et de contrôle sur la manière dont vos applications Node.js sont lancées.
Cette fonctionnalité est particulièrement utile pour les utilisateurs qui doivent spécifier des arguments, des options (flags) ou des variables d’environnement spécifiques au démarrage.

En utilisant cet egg, vous pouvez facilement configurer votre serveur pour exécuter n’importe quel script Node.js, qu’il s’agisse d’un serveur de jeu, d’un bot ou de toute autre application.
La fonctionnalité de commande de démarrage personnalisée permet un ajustement précis de l’environnement d’exécution du serveur, ce qui le rend idéal pour des cas d’utilisation plus complexes où les paramètres par défaut ne sont pas suffisants.

De plus, cet egg peut être personnalisé afin de prendre en charge des versions spécifiques de Node.js, ce qui le rend polyvalent pour différents besoins de projets. Grâce à ces options, la gestion et le déploiement d’applications Node.js via Pterodactyl deviennent simples et efficaces.

---

L’egg NodeJS pour Pterodactyl prend en charge un large éventail de versions de Node.js, vous permettant d’exécuter vos applications avec la version la plus adaptée à votre projet.  
Voici la liste des versions prises en charge :

- Node.js v12  
- Node.js v14  
- Node.js v15  
- Node.js v16  
- Node.js v17  
- Node.js v18  
- Node.js v19  
- Node.js v20  
- Node.js v21  

Avec cet egg, vous pouvez sélectionner la version de Node.js qui correspond le mieux aux besoins de votre projet, garantissant ainsi compatibilité et performances.
